By Jeremy Wilken - Software Engineer, eBay

Web Components

are the future present.

My definition

Web Componets are custom elements that behave like standard HTML elements and are easy to reuse without interfering with other elements.

Key Goals of Web Components

  • Encapsulation - keep component logic isolated
  • Isolation - keep component internals hidden from DOM
  • Reusability - make features extremely easy to reuse
  • Native - function like standard html elements with event APIs
  • Customizable - style and extend like standard html elements
  • Declarative - favor declaring features first with html

Web Components are 4 technologies

  • Custom Elements (encapsulation, isolation, native, declarative)
  • HTML Imports (reusability)
  • Templates (encapsulation)
  • Shadow DOM (isolation)

Fair warning

The Spec is not finalized, a polyfill is needed,
and API changes are possible.

Let's dig into code

Using an element

Vanilla Javascript

is using just ES5 Javascript to create web components.

Vanilla Javascript Example

https://gnomeontherun.github.io/
web-components-today/components/vanilla

Vanilla Javascript

  • Just the native APIs
  • Most flexibility
  • Can be very efficient
  • Great for simple components
  • Most work to implement
  • Most opportunity for bad design
  • Lacks many supporting tools
  • Best practices still in progress

X-Tag

is a custom elements toolkit.

X-Tag Example

https://gnomeontherun.github.io/
web-components-today/components/x-tag

X-Tag

  • Cleaner API
  • Lifecycle and events
  • Mixins and other utilities
  • Provides support for IE9+
  • Doesn't use Shadow DOM (manual)
  • Documentation could be better
  • No two way data binding
  • Templating not any different

Polymer

is a two part toolkit. It contains a set of polyfills, and a framework to develop custom elements.

Polymer Example

https://gnomeontherun.github.io/
web-components-today/components/polymer

Polymer

  • Custom elements as APIs
  • Two way data binding
  • Rich framework
  • Supported, docs are improving
  • Things are still changing
  • Loads a lot of assets
  • Alpha, use with caution
  • IE10+

AngularJS

is a Javascript toolkit for building applications in the browser.

AngularJS Example

https://gnomeontherun.github.io/
web-components-today/components/angular

Web Components using AngularJS

  • Extensive APIs
  • Very declarative markup
  • Lots of flexibility
  • Strong templating
  • No styling isolation
  • Difficult to reuse
  • Requires Angular application
  • Doesn't use native APIs

Problems Web Components do not solve

So keep these in mind as you build.

  • External dependency management (like jQuery)
  • Accessibility requirements
  • Duplication of features, the same as jQuery plugin duplication
  • Doesn't enforce quality of componentization

Other considerations

  • Packaging options: Polymer has Vulcanize, Grunt/Gulp/etc tasks
  • Favor simplicity over complexity
  • Limit dependencies
  • Make web components like native elements
